# 📚 Telegram-бот для образовательного центра

Production-ready бот на aiogram 3 для выдачи рабочих тетрадей, сдачи РТ,
чек-листов, записи на зачёты и управления учениками.

## ⚡ Запуск локально

### 1. Клонировать / распаковать проект

```bash
cd tgbot
```

### 2. Создать виртуальное окружение

```bash
python3 -m venv venv
source venv/bin/activate        # Linux/Mac
# venv\Scripts\activate.bat    # Windows
```

### 3. Установить зависимости

```bash
pip install -r requirements.txt
```

### 4. Создать .env файл

```bash
cp .env.example .env
nano .env
```

Заполнить:
```
BOT_TOKEN=ваш_токен_от_BotFather
ADMINS=ваш_telegram_id
```

Получить свой Telegram ID можно через @userinfobot.

### 5. Инициализировать базу данных

```bash
python init_db.py
```

### 6. Запустить бота

```bash
python main.py
```

## 🚀 Деплой на Railway

### 1. Создать репозиторий на GitHub

```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/ВАШ_НИК/РЕПОЗИТОРИЙ.git
git push -u origin main
```

### 2. Railway настройка

1. Зайти на [railway.app](https://railway.app)
2. New Project → Deploy from GitHub repo
3. Выбрать репозиторий
4. В разделе **Variables** добавить:
   - `BOT_TOKEN` = ваш токен
   - `ADMINS` = ваш telegram_id
5. Добавить **Volume** (обязательно для сохранения данных!):
   - Mount path: `/app` (или путь к папке проекта)
6. Деплой запустится автоматически

> ⚠️ **Важно**: На Railway обязательно подключите постоянный Volume,
> иначе SQLite база и файлы будут сбрасываться при каждом перезапуске!

## 👤 Как добавить администратора

1. Получить Telegram ID пользователя (через @userinfobot)
2. Добавить в `.env` в переменную ADMINS через запятую:
   ```
   ADMINS=123456789,987654321
   ```
3. Перезапустить бота. При следующем /start роль автоматически станет admin.

---

## 👨‍🏫 Как добавить куратора

1. Попросить человека написать `/start` боту (регистрация)
2. Открыть **⚙️ Админ-панель**
3. Нажать **👨‍🏫 Добавить куратора**
4. Ввести Telegram ID куратора
5. Ввести имя (или `+` для автоматического)

Куратор получит:
- Расширенное меню (Мои группы, Сданные РТ)
- PDF сдач своих учеников
- Уведомления о зачётах

---

## 👨‍🎓 Как добавить ученика

1. Попросить ученика написать `/start` боту
2. Открыть **⚙️ Админ-панель**
3. Сначала создать группу (**👥 Создать группу**) если её нет
4. Нажать **👨‍🎓 Добавить ученика**
5. Ввести Telegram ID, выбрать группу, ввести ФИО

Ученик автоматически получает:
- Роль `student`
- Доступ к РТ и чек-листам
- Привязку к куратору

---

## 📚 Как добавить рабочую тетрадь

1. Открыть **⚙️ Админ-панель**
2. Нажать **➕ Добавить РТ**
3. Отправить файл (PDF, DOCX, фото и т.д.)
4. Ввести название/тему

Серийный номер присваивается автоматически.
Файл сохраняется в папку `/files/`.

---

## ✅ Как добавить чек-лист

Аналогично РТ, но через **➕ Добавить чек-лист**.
Файлы хранятся в `/checklists/`.

---

## 📤 Как ученик сдаёт РТ

1. Нажать **📤 Сдать РТ** в главном меню
2. Ввести ФИО полностью
3. Отправлять фото страниц по одному
4. Нажать **📄 Отправить в PDF**

Бот:
- Собирает все фото в порядке отправки
- Конвертирует через Pillow + img2pdf в PDF
- Сохраняет в `/submissions/`
- Записывает в БД
- Отправляет PDF куратору с подписью

---

## 🗓 Как проходит запись на зачёт

**Куратор:**
1. В меню куратора нажать **🗓 Создать слоты зачёта**
2. Ввести дату, начало, конец, длительность слота, ссылку Meet
3. Бот автоматически создаёт все временны́е слоты

**Ученик:**
1. Нажать **🗓 Запись на зачёт**
2. Выбрать свободный слот из списка
3. Подтверждение придёт в чат

**Отмена:** кнопка **❌ Отменить запись** (доступна если до зачёта > 1 часа).

---

## 📊 Как работает экспорт

1. **⚙️ Админ-панель** → **📤 Экспорт сдач** (или **📤 Экспорт зачётов**)
2. Выбрать период: Сегодня / Неделя / Месяц / Всё время
3. Бот генерирует .xlsx и отправляет в чат

Используется openpyxl с автоматической шириной колонок и чередующимися цветами строк.

## ⏰ Как работает APScheduler

Scheduler запускается при старте бота и каждую минуту проверяет таблицу `exam_bookings`.

Правила уведомлений:
- **За 10 минут** (9:00–10:59 до начала): отправляется ученику и куратору
- **В момент начала** (±60 сек): отправляется обоим со ссылкой Meet
- Флаги `notified_10_min` и `notified_start` предотвращают повторную отправку

## 🔒 Безопасность и резервные копии

### Резервное копирование вручную

```bash
sqlite3 bot.db ".backup backup_$(date +%Y%m%d).db"
```

### Автоматическое копирование (cron)

```bash
crontab -e
# Добавить строку (бэкап каждый день в 03:00):
0 3 * * * /полный/путь/к/backup.sh
```

### Что бэкапить

| Что          | Где                   | Почему важно           |
|--------------|-----------------------|------------------------|
| bot.db       | корень проекта        | Вся база данных        |
| /files/      | рабочие тетради       | Загруженные файлы      |
| /checklists/ | чек-листы             | Загруженные файлы      |
| /submissions/| PDF сдач учеников     | Архив работ            |
| .env         | корень (отдельно)     | Токен и секреты        |

### Рекомендации безопасности

1. **Никогда не коммитьте .env** в Git — добавьте в `.gitignore`
2. **WAL mode** включён по умолчанию — защита от corruption при сбое
3. **PRAGMA foreign_keys=ON** — целостность данных
4. **Ролевая модель** — admin/curator/student/user с фильтрами на каждом хэндлере
5. Для Railway — используйте Volume для персистентности данных
6. Делайте бэкап перед каждым обновлением кода

## 🛠 Частые проблемы

**Бот не отвечает:**
- Проверьте BOT_TOKEN в .env
- Убедитесь что бот запущен: `python main.py`

**Файлы не сохраняются:**
- На Railway добавьте Volume
- Проверьте права на директории files/, submissions/

**APScheduler не шлёт уведомления:**
- Проверьте временну́ю зону в `scheduler_service.py` и `config.py`
- Убедитесь что в `exam_slots` правильная дата формата `YYYY-MM-DD`

**PDF не создаётся:**
- Проверьте установку: `pip install Pillow img2pdf`
- Убедитесь что фото сохраняются в /temp/ (права на запись)
